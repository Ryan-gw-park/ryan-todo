# Loop-26 목업 참조 가이드

## 목업 파일 위치
`docs/mockup-v8.html` — 프로젝트 루트에 배치

## 사용 방법
각 Loop 실행 전, 브라우저에서 `docs/mockup-v8.html`을 열고 해당 탭/모드를 확인한 후 구현에 착수한다.

---

## 목업 ↔ 작업 지시서 매핑

### Loop-26.1 (사이드바)
- 목업 좌측 사이드바 전체 확인
- 글로벌 뷰 5개 아이템 레이아웃
- 팀 프로젝트 / 개인 프로젝트 섹션 분리
- 팀 선택 드롭다운 (로고 아래)
- 하단: 도움말 + 프로필

### Loop-26.2 (프로젝트 레이어 셸)
- 사이드바에서 "정기주총" 클릭 시 메인 영역 변화 확인
- 프로젝트 헤더 (닷 + 이름 + 메타 + 탭 3개)
- Tasks 탭 클릭 → 모드 바 (목록/아웃라이너/타임라인) 표시
- 병렬 보기 탭 클릭 → 모드 바 (타임라인/결과물+Task) 표시
- Reference 탭 → 모드 바 없음 (바로 콘텐츠)

### Loop-26.3 (Reference 탭)
- Reference 탭 클릭 시 화면 확인
- 좌측: 마일스톤 목록 (컬러 닷 + 제목 + 설명 + 날짜 + D-day)
- 우측: 결과물 카드 (좌측 컬러 바 = 마일스톤 색상, 번호 + 제목 + 담당자 + 설명 + 태그 + Task 연결 수)
- 마일스톤과 결과물이 같은 행에 수평 정렬
- 결과물 없는 마일스톤 → 우측 "결과물 없음 + 추가"
- 하단 접힌 섹션: 참조 문서, 합의된 정책 (▸ 클릭으로 펼침)

### Loop-26.4 (Tasks 탭)
- Tasks 탭 > "목록" 모드
  - 상단 Reference 요약 바 (다음 마일스톤 + D-day + 미연결 결과물 수)
  - 오늘 할일 / 이번 주 / 다음 주 그룹
  - Task 옆 결과물 태그 (작은 회색 태그)
  - 담당자 컬러 배지 + 날짜
- Tasks 탭 > "아웃라이너" 모드
  - 기존 ProjectView와 100% 동일한 Workflowy 스타일
  - 펼침/접힘, 불릿포인트 계층, 키보드 단축키 전체
  - 목업에서는 아웃라이너 모드를 직접 확인할 수 없음 — **기존 앱의 프로젝트 뷰 스크린샷(Image 5)을 참조**
- Tasks 탭 > "타임라인" 모드
  - 프로젝트 내 Task만 간트 바로 표시

### Loop-26.5 (병렬 보기)
- 병렬 보기 > "타임라인" 모드
  - 전체 마일스톤 간트 (간결한 조망 뷰)
  - 임박 배지, 오늘 기준선
- 병렬 보기 > "결과물+Task" 모드
  - 마일스톤 헤더 행 (컬러 닷 + D-day)
  - 결과물별 행: 좌측 이름 + 중앙 간트 바 + 우측 연결 Task 목록
  - 미배정 행: 스트라이프 배경 + "⚠ 미배정" + "+ Task 추가" 버튼
  - 마일스톤에 결과물 없는 경우: "결과물 미정" 행

## Task 데이터 동기화 확인

프로젝트 레이어에서 생성/수정/삭제한 Task는 기존 Zustand store(`useStore`)의
`addTask`, `updateTask`, `deleteTask`, `toggleDone` 액션을 **그대로 사용**한다.

따라서:
- 프로젝트 레이어 Tasks 탭에서 Task 생성 → 글로벌 오늘할일 뷰에 즉시 반영
- 글로벌 매트릭스에서 Task 완료 → 프로젝트 레이어 Tasks 탭에 즉시 반영
- DetailPanel에서 노트 편집 → 아웃라이너 모드에 즉시 반영

**Reference 데이터(마일스톤, 결과물 등)는 별도 훅으로 관리되므로
글로벌 뷰와 직접 연동되지 않는다. 이는 의도된 설계.**
