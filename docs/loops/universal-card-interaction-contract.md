# 통합 카드 상호작용 규격 (Universal Card Interaction Contract)

> **적용 범위:** 앱 전체 — 글로벌 뷰, 프로젝트 뷰, 노트 뷰, 모달 내부 목록 어디서든
> **적용 대상:** 프로젝트 카드, 마일스톤 카드, 할일 카드 3종
> **원칙:** 같은 종류의 카드는 어디에 등장하든 동일하게 동작한다

---

## 1. 카드 영역 분리 (Hit Zone)

모든 카드는 4개 영역으로 나뉘고, 각 영역의 이벤트 핸들링이 결정적으로 다르다.

```
┌──────┬──────────────────┬─────────────────────┬──────┐
│ 상태  │    제목 텍스트     │    본문 영역          │ 상세  │
│ 영역  │                  │                     │ 아이콘│
│      │ 클릭=인라인 편집   │ 클릭=펼치기/접기      │      │
│      │                  │ 드래그=DnD 즉시 전환  │ 클릭= │
│      │                  │                     │ 상세  │
│      │                  │                     │ 진입  │
└──────┴──────────────────┴─────────────────────┴──────┘
  좌측                중앙                           우측
```

### 1-1. 상태 영역 (좌측, 28~32px)

| 카드 종류 | 표시 | 클릭 동작 |
|----------|------|---------|
| 할일 | 체크박스 | toggleDone(id) |
| 마일스톤 | 컬러 도트 | 없음 (시각 표시만) |
| 프로젝트 | 컬러 보더 (왼쪽 3px) | 없음 |

### 1-2. 제목 텍스트 영역

| 이벤트 | 동작 |
|--------|------|
| 클릭 | 인라인 편집 모드 진입 (input/textarea 전환) |
| 편집 중 Enter | 저장 후 편집 종료 |
| 편집 중 Escape | 취소, 원래 값 복원 |
| 편집 중 외부 클릭 | 저장 후 편집 종료 |

편집 결과: updateTask/updateMilestone/updateProject(id, { text/title/name })

### 1-3. 본문 영역 (제목 텍스트 외 나머지)

**클릭과 드래그를 구분하는 방법:**

```javascript
// mousedown 시 좌표 기록
// mouseup 시 이동 거리 계산
// 이동 거리 < 5px → 클릭으로 판정 → 펼치기/접기
// 이동 거리 >= 5px → 드래그로 판정 → DnD 모드

let startX, startY;

onMouseDown = (e) => {
  startX = e.clientX;
  startY = e.clientY;
  // 아직 DnD 시작하지 않음
};

onMouseMove = (e) => {
  const dx = Math.abs(e.clientX - startX);
  const dy = Math.abs(e.clientY - startY);
  if (dx + dy >= 5 && !isDragging) {
    isDragging = true;
    startDnD(); // DnD 모드 전환
  }
};

onMouseUp = (e) => {
  if (!isDragging) {
    toggleExpand(); // 클릭 → 펼치기/접기
  } else {
    endDnD(); // 드롭
  }
};
```

**펼치기/접기 시 표시되는 내용:**

| 카드 종류 | 펼침 내용 |
|----------|---------|
| 할일 | 노트 미리보기 (2줄), 기간, 댓글 수, MS 뱃지 |
| 마일스톤 | 하위 할일 목록 (각각 할일 카드 규격 적용) |
| 프로젝트 | 하위 MS 목록 (각각 MS 카드 규격 적용) |

### 1-4. 상세 아이콘 (우측, 36px)

| 카드 종류 | 아이콘 | 클릭 동작 |
|----------|-------|---------|
| 할일 | ▶ 또는 ··· | DetailPanel (사이드 패널) 열기 |
| 마일스톤 | ▶ 또는 ··· | 마일스톤 상세 모달 열기 |
| 프로젝트 | ▶ 또는 ··· | 프로젝트 설정 모달 열기 |

아이콘은 평소 숨김, 카드 호버 시 표시 (모바일에서는 항상 표시).

## 2. DnD 상세 규격

### 2-1. DnD 시각 피드백 (모든 카드 공통)

| 상태 | 시각 효과 |
|------|---------|
| 드래그 시작 | 원본 카드: opacity 0.3, 점선 테두리 (자리 표시) |
| 드래그 중 | 커서를 따라다니는 카드 축소 복제본 (DragOverlay) |
| 유효 드롭 대상 위 | 대상 영역: 파란 테두리 + 연한 파란 배경 |
| 무효 드롭 대상 위 | 변화 없음, 커서 not-allowed |
| 드롭 완료 | 즉시 위치 반영 (낙관적 업데이트) |
| 드롭 취소 (Escape) | 원본 위치로 복귀 |

### 2-2. 카드별 DnD 가능 동작

**할일 카드:**

| 드롭 대상 | 변경 필드 | 적용 뷰 |
|----------|---------|---------|
| 다른 프로젝트 영역 | { projectId } | 오늘 할일, 매트릭스, 타임라인 |
| 다른 MS 그룹 | { keyMilestoneId } | 전체 할일, 프로젝트 뷰 |
| 다른 카테고리 행 | { category } | 매트릭스 |
| 다른 담당자 행 | { assigneeId } | 매트릭스, 주간 플래너 |
| 다른 요일 셀 | { dueDate } | 주간 플래너 |
| 백로그 사이드바 | { dueDate: null, category: 'backlog' } | 주간 플래너 |
| 같은 영역 내 순서 | reorderTasks() | 모든 뷰 |

**마일스톤 카드:**

| 드롭 대상 | 변경 필드 | 적용 뷰 |
|----------|---------|---------|
| 다른 담당자 행 | updateMilestone({ owner_id }) | 매트릭스 MS 모드 |
| 같은 프로젝트 내 순서 | reorderMilestones() | 프로젝트 뷰 |
| 다른 프로젝트 | **불가** (MS는 프로젝트 귀속) | — |

**프로젝트 카드:**

| 드롭 대상 | 변경 필드 | 적용 뷰 |
|----------|---------|---------|
| 순서 변경 | reorderProjects() | 사이드바 |
| 다른 영역 이동 | **불가** (프로젝트는 최상위) | — |

### 2-3. DnD 중 드롭 대상 하이라이트 규칙

드래그하는 카드 종류에 따라 유효한 드롭 대상이 다르다:

```javascript
// 할일 카드 드래그 중:
//   유효: 프로젝트 영역, MS 그룹, 카테고리 행, 담당자 행, 요일 셀, 백로그
//   무효: 프로젝트 헤더 자체, MS 헤더 자체 (할일이 프로젝트/MS가 될 수 없음)

// MS 카드 드래그 중:
//   유효: 담당자 행 (매트릭스 MS 모드), 같은 프로젝트 내 MS 순서
//   무효: 다른 프로젝트 열, 카테고리 행, 요일 셀

// 프로젝트 카드 드래그 중:
//   유효: 프로젝트 순서 변경 (사이드바)
//   무효: 대부분
```

## 3. 카드 메타 정보 레이아웃

### 3-1. 할일 카드 메타 행

```
□ 안건 PPT - 수요일 오후 3시 Jason                  ▶
  [● 안건자료PPT]  Ryan  3/17~3/20
```

- 1행: 체크박스 + 제목 + 상세 아이콘
- 2행: MS 뱃지 (있으면) + 담당자 (있으면) + 기간 (있으면)
- 강조 색상: 카드 배경 또는 왼쪽 보더

### 3-2. MS 카드 메타 행

```
● 공증 준비                                         ▶
  ▇▇▇░░ 33% (2/6)  Ryan  2/1~3/10
```

- 1행: 컬러 도트 + 제목 + 상세 아이콘
- 2행: 진행률 바 + 할일 수 + 담당자 + 기간

### 3-3. 프로젝트 카드 메타 행

```
┃ 정기주총                                          ▶
┃  11건  42%  Ryan  1/15~3/28
```

- 왼쪽 컬러 보더
- 1행: 이름 + 상세 아이콘
- 2행: 할일 수 + 진행률 + 오너 + 기간

## 4. 현재 뷰별 불일치 → 통합 후 변경점

### 4-1. TodayView

| 항목 | 현재 | 통합 후 |
|------|------|--------|
| 인라인 편집 | input 항상 편집 가능 | 제목 클릭 시에만 편집 |
| 상세 진입 | ▶ 호버 아이콘 | ▶ 아이콘 (동일) |
| DnD | 프로젝트 간 이동 | 동일 |
| 펼치기 | 없음 | **추가**: 본문 클릭 시 노트/기간 표시 |
| MS 뱃지 | 없음 | **추가** |

### 4-2. AllTasksView

| 항목 | 현재 | 통합 후 |
|------|------|--------|
| 인라인 편집 | 없음 (읽기전용) | **추가**: 제목 클릭 시 편집 |
| 상세 진입 | 행 전체 클릭 | ▶ 아이콘으로 변경 |
| DnD | 없음 | **추가**: MS 간 이동, 프로젝트 간 이동 |
| 펼치기 | 없음 | **추가**: MS 그룹핑 모드에서 MS 펼치기 |

### 4-3. MatrixView / TeamMatrixView

| 항목 | 현재 | 통합 후 |
|------|------|--------|
| 인라인 편집 | 텍스트 클릭→textarea | 제목 클릭→input (동일 원리) |
| 상세 진입 | ▶ 호버 아이콘 | ▶ 아이콘 (동일) |
| DnD | 카테고리/담당자/프로젝트 | 동일 + MS 뱃지 추가 |
| 펼치기 | 없음 | **추가**: 본문 클릭 시 노트/기간 표시 |

### 4-4. CompactTaskList (ProjectView)

| 항목 | 현재 | 통합 후 |
|------|------|--------|
| 인라인 편집 | 텍스트 클릭→input | 동일 |
| 상세 진입 | ▶ 아이콘 | 동일 |
| DnD | MS 간 이동 | 동일 |
| 노트 토글 | 별도 노트 아이콘 | 본문 클릭 = 펼치기로 통합 |

### 4-5. TimelineView

| 항목 | 현재 | 통합 후 |
|------|------|--------|
| 좌측 패널 행 | 이름만 표시 | 통합 카드 규격 적용 (축소 버전) |
| 상세 진입 | 바 클릭 (이동량<5px) | 좌측 패널 ▶ 아이콘 |
| DnD | 간트바 드래그 (기간) | 간트바 = 기간, 좌측 패널 = 프로젝트/MS 이동 |

### 4-6. 주간 플래너 (신규)

처음부터 통합 카드 규격으로 구현.

### 4-7. 모달 내부 목록

| 위치 | 적용 |
|------|------|
| 프로젝트 설정 모달 → MS 목록 | MS 카드 규격 (제목 편집, 펼치기, 상세 아이콘) |
| MS 상세 모달 → 할일 목록 | 할일 카드 규격 (체크박스, 제목 편집, 상세 아이콘) |

## 5. 공통 컴포넌트 설계

### 5-1. UniversalCard

```jsx
<UniversalCard
  type="task" | "milestone" | "project"
  data={item}                    // raw 데이터
  expanded={boolean}
  onToggleExpand={() => ...}
  onTitleEdit={(newText) => ...}
  onStatusToggle={() => ...}     // 할일: toggleDone, MS: 상태변경
  onDetailOpen={() => ...}       // 상세 패널/모달 열기
  draggable={boolean}
  onDragStart={() => ...}
  renderMeta={() => ...}         // 뷰별 커스텀 메타 (MS 뱃지, 담당자 등)
  renderExpanded={() => ...}     // 펼침 시 내용 (뷰별 커스텀)
  compact={boolean}              // 타임라인 좌측 패널 등 축소 모드
/>
```

### 5-2. 내부 구조

```
UniversalCard
├── StatusZone        (좌측: 체크박스/컬러도트/컬러보더)
├── TitleZone         (제목: 클릭→인라인 편집)
├── BodyZone          (본문: 클릭→펼치기, 드래그→DnD)
│   ├── MetaRow       (MS 뱃지, 담당자, 기간 등)
│   └── ExpandedContent (펼침 시 하위 목록)
└── DetailZone        (우측: ▶ 아이콘)
```

### 5-3. DnD 통합

```
useDnD(cardType, data)
├── drag source 설정 (카드 자체)
├── drag overlay 렌더링 (축소 복제본)
├── drop target 판별 (카드 종류에 따라 유효 대상 결정)
└── drop 시 updateTask/updateMilestone/updateProject 호출
```

dnd-kit 기반. 모든 뷰에서 같은 DnD context를 공유하되, 뷰별로 droppable 영역만 다르게 정의.

## 6. 구현 우선순위

이 규격은 **Loop-35A 이전에 선행 작업**으로 처리해야 한다.
공통 컴포넌트가 없으면 각 뷰를 개별 수정하게 되어 일관성이 깨진다.

### 선행: UniversalCard 컴포넌트 추출 (Loop-35 전)

1. UniversalCard 기본 구조 (4영역 분리)
2. 인라인 편집 (클릭→편집, Enter/Escape/blur 처리)
3. 펼치기/접기 (클릭 vs 드래그 5px 판별)
4. DnD 소스 설정 (dnd-kit draggable)
5. 상세 아이콘 (호버 시 표시)
6. 3종 카드별 StatusZone/MetaRow 분기

### 적용 순서

7. TodayView에 UniversalCard 적용
8. AllTasksView에 UniversalCard 적용
9. MatrixView/TeamMatrixView에 UniversalCard 적용
10. CompactTaskList에 UniversalCard 적용
11. TimelineView 좌측 패널에 UniversalCard(compact) 적용
12. 모달 내부 목록에 UniversalCard 적용
13. 주간 플래너 신규 뷰에서 UniversalCard 사용
