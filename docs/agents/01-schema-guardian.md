# Agent 01: Schema Guardian

> Protects DB schema integrity: tables, columns, migrations, RLS activation.
> Baseline: 2026-03-18 (codebase diagnostic)

---

## BLOCK Rules

### B1. Never rename existing tables or columns

These names are immutable. `ALTER TABLE RENAME` is forbidden.

```
Legacy tables: tasks, projects, memos, push_subscriptions, ui_state
Legacy columns (tasks): id, text, done, category, alarm, notes, project_id,
                        sort_order, created_at, prev_category, due_date, start_date
Legacy columns (projects): id, name, color, sort_order
Legacy columns (memos): id
```

### B2. New columns MUST have DEFAULT

```sql
-- Correct
ALTER TABLE tasks ADD COLUMN new_field text DEFAULT '';

-- Forbidden — existing rows get NULL, may break NOT NULL or app logic
ALTER TABLE tasks ADD COLUMN new_field text;
```

### B3. PK type rules

| Table type | PK type | Generated by |
|-----------|---------|-------------|
| Legacy (tasks, projects, memos) | `text` | Frontend |
| New | `uuid DEFAULT gen_random_uuid()` | DB |

When new tables reference legacy tables: use `text` column, no DB FK, app-level reference only.

### B4. DDL goes in supabase/migrations/ as new files only

```bash
# Correct — new timestamped file
supabase/migrations/20260318120000_add_new_feature.sql

# Forbidden — editing existing file breaks migration chain checksum
supabase/migrations/20250101000000_initial.sql  ← NEVER edit
```

### B5. RLS is mandatory

Every new table MUST have RLS enabled + at least one SELECT policy in the same migration file.

```sql
ALTER TABLE new_table ENABLE ROW LEVEL SECURITY;
CREATE POLICY "new_table_select" ON new_table FOR SELECT USING (...);
```

### B6. Verify column existence before creating indexes

Index target column must actually exist. Check ordering within the migration.

```sql
-- Correct — column first, then index
ALTER TABLE notifications ADD COLUMN IF NOT EXISTS updated_at timestamptz DEFAULT now();
CREATE INDEX IF NOT EXISTS idx_notifications_updated ON notifications (updated_at);

-- Forbidden — index on nonexistent column = migration error
CREATE INDEX idx_notifications_updated ON notifications (updated_at);
```

### B7. Respect CHECK constraints

`tasks` table has a `valid_scope` CHECK:

```
private:  team_id IS NULL     AND assignee_id IS NULL
team:     team_id IS NOT NULL AND assignee_id IS NULL
assigned: team_id IS NOT NULL AND assignee_id IS NOT NULL
```

`projects` table has a `project_ownership` CHECK:

```
team project:     team_id IS NOT NULL AND user_id IS NULL
personal project: team_id IS NULL     AND user_id IS NOT NULL
```

Violating these fails INSERT/UPDATE silently on the DB side.

## Known Divergences

| ID | Severity | Description |
|----|----------|-------------|
| KD-1.1 | **HIGH** | Migration chain broken: `ref_*` → `key_*` rename lives in `docs/migrations/` only, not in `supabase/migrations/`. Fresh deploy will have broken schema |
| KD-1.2 | MEDIUM | `memos` table missing `set_updated_at` trigger. `updated_at` frozen at creation time |
| KD-1.3 | MEDIUM | `team_members` table missing `updated_at` column. Prerequisite for adding to SYNC_TABLES |
| KD-1.4 | LOW | `push_subscriptions.user_id` is `text` type (not uuid). RLS uses `::uuid` cast as workaround |
| KD-1.5 | LOW | `comments.task_id` and `notifications.task_id` have no FK constraint (intentional, but orphan records possible) |

## Verification Commands

```bash
# Detect rename attempts on existing tables
grep -rn "RENAME\s\+COLUMN\|RENAME\s\+TABLE\|ALTER.*RENAME" supabase/migrations/ --include="*.sql"

# Detect ADD COLUMN without DEFAULT
grep -n "ADD COLUMN" supabase/migrations/*.sql | grep -v "DEFAULT\|IF NOT EXISTS"

# Detect CREATE TABLE without ENABLE ROW LEVEL SECURITY in same file
```
